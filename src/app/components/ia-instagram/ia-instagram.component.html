El proyecto

Un pipeline de IA que trabaja mientras duermes

Esta demo muestra cómo construir un sistema completamente automatizado capaz de generar, diseñar y publicar Instagram Stories a diario sin ninguna intervención humana. El flujo combina herramientas de IA generativa para crear el texto y la imagen, y n8n como motor de orquestación para encadenar cada paso en el momento exacto.

No se trata de programar stories manualmente; se trata de enseñarle al sistema qué contar y dejar que él decida cómo y cuándo contarlo.

Bajo el capó

Anatomía del workflow n8n

Schedule Trigger — Inicio automático
El nodo Schedule de n8n dispara el flujo a la hora configurada. Soporta CRON estándar, lo que permite programar la publicación en la franja horaria de mayor actividad de la audiencia.
Set Node — Contexto del día
Se inyectan variables dinámicas: fecha, tema del día, tono de comunicación y cualquier parámetro que personalice el contenido. Esto permite variar el mensaje cada día sin modificar el flujo.
HTTP Request → OpenAI — Generación de texto
Una llamada HTTP con el prompt construido dinámicamente solicita al modelo LLM el texto de la story. La respuesta se parsea en el siguiente nodo para extraer titular y hashtags por separado.
HTTP Request → API de Imagen — Creación visual
Con el texto como input, se construye un prompt de imagen y se llama a la API generativa. La URL de la imagen resultante se almacena en el contexto del workflow.
HTTP Request → Instagram Graph API — Publicación
Dos llamadas encadenadas: primero se crea el contenedor del media con la URL de la imagen, y en segundo lugar se publica con el endpoint de publicación. El token OAuth se gestiona de forma segura desde las credenciales de n8n.
Google Sheets / Airtable — Log de ejecución
Cada publicación queda registrada con timestamp, ID del post publicado y status de la llamada API. Permite auditar el histórico y detectar fallos en tiempo real.
